Integration Guide
Complete playbook for building the syllabus materials module — from Google Drive setup to in-portal PDF rendering, download links, and the admin management screen.

01 — Concept
How it works
The portal never stores files. It stores only a Google Drive file ID — a short string like 1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms. All file delivery — preview, stream, download — comes directly from Google's servers. You manage files in Drive, your portal just points to them.
System flow
1
ANO uploads file to Google Drive
PDF, video, or document uploaded to the unit's shared Drive folder. Sharing is set to "Anyone with the link — Viewer". This takes 60 seconds.
2
ANO pastes the share URL into the admin portal
The portal extracts the file ID from the pasted URL automatically. The ANO fills in the title, subject, and wing. No technical knowledge required.
3
Backend validates the link
The server makes a HEAD request to the Drive preview URL. If it returns 200, the link is accessible. If it redirects to accounts.google.com, it's restricted — the material is flagged before cadets ever see it.
4
Material is saved to the database
Only the Drive file ID is stored — never a full URL. The full URL is constructed at runtime from the ID. This means if Google ever changes their URL format, you update one utility function, not thousands of database rows.
5
Cadets see and access materials
PDFs embed directly in the portal via iframe. Videos and documents open in Google's viewer in a new tab. Every material card has a Download button that triggers a direct download from Drive.
Why not store files on your server? File storage requires a CDN or object storage service (S3, Cloudflare R2), egress costs, backup strategy, and significant added complexity. For an MVP with one unit, Drive is free, reliable, globally cached, and zero maintenance. Migrate to S3 only if the unit grows significantly or Drive's access-control limitations become a real problem.
02 — Concept
Drive URL anatomy
One file ID generates three distinct URLs, each serving a different purpose. Understanding this is the foundation of the entire module.
Share URL (what Drive gives you)
https://drive.google.com/file/d/1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms/view?usp=sharing
Preview URL (iframe embed — PDFs only)
https://drive.google.com/file/d/1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms/preview
Download URL (direct download — all file types)
https://drive.google.com/uc?export=download&id=1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms

Query parameters

04 — Backend
Database schema
A single Material model added to your existing Prisma schema. It connects to User (uploader) and optionally to College for scoped visibility.
schema.prisma
model Material {
  id           String       @id @default(cuid())
  title        String
  subject      String       // e.g. "NCC Drill", "Map Reading", "First Aid"
  description  String?
  driveFileId  String       // ONLY the file ID — never the full URL
  fileType     FileType     // PDF | VIDEO | DOCUMENT
  wing         Wing?        // ARMY | NAVY | AIR_FORCE | null = all wings
  collegeId    String?      // null = visible to all colleges (Super Admin only)
  college      College?     @relation(fields: [collegeId], references: [id])
  uploadedById String
  uploadedBy   User         @relation(fields: [uploadedById], references: [id])
  accessStatus AccessStatus @default(VERIFIED)
  isActive     Boolean      @default(true)
  createdAt    DateTime     @default(now())
  updatedAt    DateTime     @updatedAt
}

enum FileType {
  PDF
  VIDEO
  DOCUMENT
}

enum AccessStatus {
  VERIFIED    // Drive returns 200 — cadets can access
  RESTRICTED  // Drive redirects to login — file is private
  PENDING     // Not yet validated
  ERROR       // Unexpected response from Drive
}
Why accessStatus? This field is what protects cadets from broken links. When validation runs (on save, and on a nightly cron), it updates this field. Cadets only see materials where accessStatus === 'VERIFIED'. Admins see all materials with the status clearly labelled.

routes/materials.ts
// POST /api/materials — create a new material
router.post('/', requireRole(['COLLEGE_ADMIN', 'SUPER_ADMIN']), async (req, res) => {
  const { title, subject, description, driveUrl, fileType, wing } = req.body

  // 1. Extract file ID from whatever URL format Drive gives
  const driveFileId = extractDriveId(driveUrl)
  if (!driveFileId) {
    return res.status(400).json({ message: 'Invalid Google Drive URL' })
  }

  // 2. Validate that the file is publicly accessible
  const accessStatus = await validateDriveAccess(driveFileId)

  // 3. Save to DB — use the requester's collegeId from JWT for ANO
  const material = await prisma.material.create({
    data: {
      title, subject, description, driveFileId, fileType, wing,
      collegeId: req.user.role === 'SUPER_ADMIN' ? null : req.user.collegeId,
      uploadedById: req.user.id,
      accessStatus,
    }
  })

  // 4. Warn the ANO if the file is restricted, but still save
  res.status(201).json({
    material,
    warning: accessStatus === 'RESTRICTED'
      ? 'File appears to be restricted. Cadets will not be able to access it until you update the sharing settings in Google Drive.'
      : null
  })
})
Cadet-side query — scoped visibility
// GET /api/materials — cadets only see what they're eligible for
router.get('/', requireAuth(), async (req, res) => {
  const { subject, fileType, wing } = req.query
  const { user } = req

  const where = {
    isActive: true,
    accessStatus: 'VERIFIED',
    // Material is visible if: no college restriction OR matches cadet's college
    OR: [
      { collegeId: null },         // global material
      { collegeId: user.collegeId }, // college-specific material
    ],
    // Wing filter: show if no wing restriction OR matches cadet's wing
    ...(user.wing ? { OR: [{ wing: null }, { wing: user.wing }] } : {}),
    // Optional client-side filters
    ...(subject  ? { subject }  : {}),
    ...(fileType ? { fileType } : {}),
  }

  const materials = await prisma.material.findMany({
    where,
    select: {
      id: true, title: true, subject: true,
      description: true, fileType: true, wing: true,
      driveFileId: true,  // needed to construct URLs client-side
      createdAt: true,
    },
    orderBy: { createdAt: 'desc' }
  })

  res.json(materials)
})
06 — Backend
Link validation
The most important backend function in this module. It protects every cadet from clicking a link that silently redirects them to a Google login page.
utils/drive.ts
import axios from 'axios'

// ── ID extraction ─────────────────────────────────────────────
export function extractDriveId(url: string): string | null {
  const patterns = [
    // /file/d/ID/view  (most common share link format)
    /\/file\/d\/([a-zA-Z0-9_-]{25,})/,
    // ?id=ID  (older download links)
    /[?&]id=([a-zA-Z0-9_-]{25,})/,
    // /d/ID  (shortened links)
    /\/d\/([a-zA-Z0-9_-]{25,})/,
  ]
  for (const pattern of patterns) {
    const match = url.match(pattern)
    if (match) return match[1]
  }
  return null
}

// ── URL construction ──────────────────────────────────────────
export const driveUrl = {
  preview:  (id: string) => `https://drive.google.com/file/d/${id}/preview`,
  view:     (id: string) => `https://drive.google.com/file/d/${id}/view`,
  download: (id: string) => `https://drive.google.com/uc?export=download&id=${id}`,
}

// ── Access validation ─────────────────────────────────────────
export async function validateDriveAccess(
  fileId: string
): Promise<'VERIFIED' | 'RESTRICTED' | 'ERROR'> {
  try {
    const response = await axios.get(driveUrl.preview(fileId), {
      maxRedirects: 0,       // don't follow redirects — a redirect IS the error signal
      timeout: 5000,
      validateStatus: (s) => s < 400,
    })

    const finalUrl = response.request?.res?.responseUrl ?? ''

    // If Drive redirected to accounts.google.com — file is restricted
    if (finalUrl.includes('accounts.google.com')) {
      return 'RESTRICTED'
    }

    // 200 or 204 with no redirect = file is public
    if (response.status === 200) return 'VERIFIED'

    return 'ERROR'
  } catch (err) {
    // Network failure, timeout, or 404 — treat as ERROR not RESTRICTED
    console.error('Drive validation failed:', err)
    return 'ERROR'
  }
}
Nightly revalidation cron
// cron/revalidateMaterials.ts — run nightly at 2am
import cron from 'node-cron'

cron.schedule('0 2 * * *', async () => {
  const materials = await prisma.material.findMany({
    where: { isActive: true },
    select: { id: true, driveFileId: true }
  })

  for (const m of materials) {
    const status = await validateDriveAccess(m.driveFileId)
    await prisma.material.update({
      where: { id: m.id },
      data: { accessStatus: status }
    })
  }

  console.log(`Revalidated ${materials.length} materials`)
})
Install node-cron: npm install node-cron @types/node-cron. Register the cron in your Express app entry point — import './cron/revalidateMaterials'. It fires once on server start and then every night at 2am. For a production deployment on a service like Railway or Render, a cron job is better than an in-process scheduler — use their native cron feature instead.

// Detect file type from a pasted URL if the ANO doesn't specify
export function detectFileType(url: string): 'PDF' | 'VIDEO' | 'DOCUMENT' | null {
  const lower = url.toLowerCase()
  if (lower.includes('.pdf'))  return 'PDF'
  if (lower.includes('.mp4') || lower.includes('.mov')) return 'VIDEO'
  if (lower.includes('.docx') || lower.includes('.pptx')) return 'DOCUMENT'
  return null  // ANO must select manually
}

MaterialForm.tsx — URL input handler
// components/materials/MaterialForm.tsx
import { useState } from 'react'
import { extractDriveId, detectFileType } from '@/utils/drive'

function MaterialForm({ onSave }) {
  const [driveUrl, setDriveUrl] = useState('')
  const [fileId, setFileId]   = useState<string | null>(null)
  const [idError, setIdError] = useState(false)

  // Run on every keystroke — extract ID immediately on paste
  const handleUrlChange = (val: string) => {
    setDriveUrl(val)
    const extracted = extractDriveId(val)
    setFileId(extracted)
    setIdError(val.length > 10 && !extracted)
  }

  return (
    <div>
      <label>Paste Drive share URL</label>
      <input
        value={driveUrl}
        onChange={e => handleUrlChange(e.target.value)}
        placeholder="https://drive.google.com/file/d/..."
        className={fileId ? 'border-green-400' : idError ? 'border-red-400' : ''}
      />

      {fileId && (
        <p className="text-xs text-green-700 mt-1">
          ✓ File ID extracted: {fileId}
        </p>
      )}
      {idError && (
        <p className="text-xs text-red-600 mt-1">
          Could not extract a file ID from this URL. Paste the full share link from Google Drive.
        </p>
      )}
    </div>
  )
}

export function MaterialCard({ material, onView }) {

          {material.subject} · {fileTypeLabel[material.fileType]}
        </p>
      </div>

      <div className="border-t p-2 flex gap-2">
        {isInline ? (
          <button
            onClick={() => onView(material)}
            className="flex-1 h-7 text-xs font-medium bg-navy-wash text-navy-mid rounded"
          >View</button>
        ) : (
          <a
            href={viewHref} target="_blank" rel="noreferrer"

          className="flex-1 h-7 text-xs font-medium bg-stone-mid text-ink-2 rounded flex items-center justify-center gap-1"

      </div>
    </div>
  )

iframe + Google login: A cadet who is signed into a Google account that does not have permission to view the file will see a blank iframe — no error, no message. 11 — Operations
ANO Upload guide
What you tell every ANO, word for word. Keep this as a printed SOP pinned in the unit office.
1
Go to drive.google.com
Sign in with your unit Google account (not a personal account). Navigate to the NCC Tirupati shared folder. Create a subfolder for the subject if it doesn't already exist.
2
Upload the file
Drag and drop the PDF, video, or document into the folder. Wait for the upload to complete — you'll see a green tick. Keep files under 20MB where possible. Compress PDFs using Smallpdf or ilovepdf.com before uploading if they're large.
3
Set sharing to "Anyone with the link"
Right-click the file → Share → Change from "Restricted" to Anyone with the link. Role must be Viewer (not Editor or Commenter). Click Copy link. This is the critical step — if sharing is wrong, cadets get a blank page.
4
Open the portal and add the material
Go to Materials → Add material. Paste the copied link into the Drive URL field. Fill in the title, subject, file type, and wing. The portal will automatically check if the file is accessible before saving.
5
Verify with a cadet account
Log in as a test cadet in a different browser (or Incognito). Open the material and confirm you can view and download it. This takes 2 minutes and prevents dozens of support requests.
Drive folder structure recommendation
Organise Drive as: NCC Tirupati / [Wing] / [Subject] / [Year] / filename.pdf
Example: NCC Tirupati / Army / Drill / 2026 / drill-manual-part-2.pdf
This makes it easy to find and replace outdated materials when syllabi change.
12 — Operations
Known gotchas
Every issue you will encounter, documented before you hit them.
ProblemSymptomFix
iframe blank — cadet signed into wrong Google account White iframe, no error message Always show "Can't see? Open in Drive" fallback. Ensure sharing is "Anyone with link", not "Specific people".
Large file download warning page Download button shows a Google "too large to scan" HTML page instead of the file Keep files under 25MB. Compress PDFs first. For videos, use compressed MP4 at 720p.
ANO accidentally sets sharing to "Restricted" Portal shows RESTRICTED badge on the material Validation catches this on save. ANO must fix in Drive then click Revalidate in the portal.
Drive changes the URL format ID extraction fails for new uploads Update extractDriveId() regex patterns in one place. All existing IDs in DB are unaffected.
Cadet on mobile — iframe too small PDF is unreadable on a phone screen On mobile (pointer: coarse), replace the iframe with a full-screen "Open in Drive" button instead. Drive's mobile app handles PDF rendering better than an iframe.
Video embedded in iframe — autoplay blocked Video doesn't play, browser shows permission error Videos must use the /view URL and open in a new tab. Never embed videos in an iframe.
Nightly cron marks a temporarily down file as RESTRICTED Material disappears from cadet view during Drive outage Change cron logic: only update from VERIFIED to RESTRICTED if validation fails 3 consecutive nights. Store consecutiveFailures Int @default(0) on the model.
